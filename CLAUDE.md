# FlowChain MVP Instructions

Read prompt.md before starting.

Create a single-page web application called FlowChain.

Requirements:

- Use only HTML, CSS and JavaScript.
- Everything must be contained in a single file called flowchain.html.
- Use inline CSS.
- Use inline JavaScript.
- Do not use external libraries.
- Do not use a backend.
- Do not use a database.
- Do not use real APIs.

The application should help importing SMEs reduce settlement friction and understand working-capital needs.

Features to implement:

- Supplier invoice input
- Transport cost input
- Import duty input
- VAT percentage input
- Expected sales revenue input
- Current cash balance input
- Sales period input

Calculations:

- Settlement plan
- VAT reserve
- Total obligations
- Projected cash gap
- Financing recommendation
- Estimated monthly repayment

The interface should look clean, professional and investor-oriented rather than like a school calculator.

After building the application:

1. Test the calculation logic and fix any errors.
2. Improve the UI.
3. Clearly distinguish between implemented functionality and simulated integrations.
4. Add a section explaining which external integrations are simulated and which would require real partners, API access or licenses.

Simulated integrations:

- Open Banking API
- Supplier API
- Transport API
- Tax/Customs API
- FX Provider

The MVP demonstrates the FlowChain concept and does not execute real payments, customs filings, foreign exchange transactions or financing approvals.
